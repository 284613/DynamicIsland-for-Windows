## 1. 项目概述

**DynamicIsland for Windows** 是一个 C++/Win32 桌面应用程序，旨在复刻 iPhone 14 Pro 的"灵动岛"交互体验。

### 核心特性

| 特性 | 描述 |
|------|------|
| 音乐可视化 | 监控 Spotify/网易云音乐播放，显示专辑封面、进度条、歌词 |
| 通知提醒 | 捕获第三方应用 Toast 通知，以 Alert 形式展示 |
| 网络状态 | 监控 WiFi/蓝牙连接状态变化，自动弹出提醒 |
| 文件拖放 | 支持将文件拖放到悬浮球上暂存 |
| 番茄钟 | 内置计时器，专注/休息交替 |
| 天气插件 | 显示当前天气和温度 |
| 系统监控 | 电量、充电状态监控 |
| 多Island模式 | 支持 Main/Mini/Pomodoro/Stats 四种 Island 类型 |
| 设置面板 | 提供丰富的自定义配置选项 |

## 4. 核心架构解析

### 4.1 入口点与初始化流程

**文件**: `main.cpp`

```cpp
// 入口函数
int WINAPI wWinMain(HINSTANCE hInstance, HINSTANCE hPrevInstance,
                    PWSTR pCmdLine, int nCmdShow)
{
    // 1. 初始化 WinRT 公寓
    winrt::init_apartment(winrt::apartment_type::single_threaded);

    // 2. 初始化 OLE (用于拖放功能)
    OleInitialize(nullptr);

    // 3. 设置 DPI 感知
    SetProcessDpiAwarenessContext(DPI_AWARENESS_CONTEXT_PER_MONITOR_AWARE_V2);

    // 4. 创建并运行 DynamicIsland 实例
    DynamicIsland app;
    app.Initialize();
    return app.Run();
}
```

**初始化序列**:
1. `DynamicIsland::Initialize()` — 创建窗口、初始化渲染引擎、启动各监控线程
2. `DynamicIsland::Run()` — 进入消息循环

### 4.5 弹簧物理动画系统

**文件**: `Spring.h`

使用弹簧物理模型实现平滑的尺寸过渡动画：

```cpp
class Spring
{
public:
    Spring(float tension = 180.f, float damping = 12.f)
        : m_tension(tension), m_damping(damping) {}

    // 更新动画，返回是否仍在运动
    bool Update(float dt);

    // 设置目标值
    void SetTarget(float target) { m_target = target; }

    float GetValue() const { return m_value; }

private:
    float m_value{0};        // 当前值
    float m_target{0};      // 目标值
    float m_velocity{0};    // 当前速度
    float m_tension{180.f}; // 弹簧张力
    float m_damping{12.f};   // 阻尼系数
};
```

**物理模型**:
- 受力 = 张力 × (目标值 - 当前值) - 阻尼 × 速度
- 速度 += 受力 × dt
- 位置 += 速度 × dt

两个独立的 Spring 实例分别控制 Island 的**宽度**和**高度**：

```cpp
Spring m_widthSpring;   // 宽度动画
Spring m_heightSpring;  // 高度动画
```

在 `WM_TIMER` 中 (~16ms 一次):
```cpp
m_widthSpring.Update(dt);
m_heightSpring.Update(dt);
InvalidateRect(m_hwnd, nullptr, FALSE);
```

### 5.2 MediaMonitor 媒体监控

**文件**: `MediaMonitor.cpp/h`

使用 **Windows.Media.Control** (WinRT) 和 **WASAPI** 两种方案监控媒体播放。

#### 架构

```cpp
class MediaMonitor
{
    // WinRT 方案 - GlobalSystemMediaTransportControlsSessionManager
    winrt::GlobalSystemMediaTransportControlsSessionManager m_sessionManager;

    // 背景轮询线程
    std::thread m_pollingThread;
    std::atomic<bool> m_running{true};

    // 音频会话监控 (备用)
    wil::com_ptr<IMMDeviceEnumerator> m_deviceEnumerator;
    wil::com_ptr<IAudioSessionManager2> m_audioSessionManager;
};
```

#### 监控流程

1. **初始化 WinRT Session Manager**:
   ```cpp
   GlobalSystemMediaTransportControlsSessionManager::GetAsync()
   ```

2. **注册媒体状态回调**:
   ```cpp
   sessionManager.add_SessionChanged(
       [this](auto&&, GlobalSystemMediaTransportControlsSessionChangedEventArgs args) {
           // 媒体会话变化 (播放/暂停/切歌)
       }
   );
   ```

3. **后台线程轮询** (~500ms):
   - 检查当前播放曲目
   - 获取专辑封面 (bytes → D2D Bitmap)
   - 获取播放进度

#### 获取专辑封面

```cpp
winrt::array_view<const uint8_t> albumArt = session.GetArtwork();
if (albumArt.size() > 0) {
    // bytes → IWICBitmap → ID2D1Bitmap
}
```

#### 音乐控制

```cpp

session.GetTimelineProperties().StartTime() // 进度条用
```

#### LRC 格式解析

```
[00:12.34]这是一行歌词
[mm:ss.xx]时间戳  歌词内容
```

```cpp
struct LyricLine {
    int minutes;
    int seconds;
    int milliseconds;
    std::wstring text;
};
```

#### 时间戳同步

歌词按时间戳排序，当前歌词行 = 找到 `<= currentTime` 的最后一行。

#### 缓存机制

```
x64/Release/lyrics_cache/
├── {md5(song+artist)}.lrc    # 缓存文件
└── ...
```

首次获取后缓存到本地，下次直接读取文件。

```cpp
class EventBus
```
