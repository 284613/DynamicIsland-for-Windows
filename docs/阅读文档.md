## 1. 项目概述

**DynamicIsland for Windows** 是一个 C++/Win32 桌面应用程序，旨在复刻 iPhone 14 Pro 的"灵动岛"交互体验。整体约 **13,300+ 行代码**，分布在 17 个 cpp 文件和 25 个头文件中。

### 核心特性

| 特性 | 描述 |
|------|------|
| 音乐可视化 | 监控 Spotify/网易云音乐播放，显示专辑封面、进度条、歌词 |
| 通知提醒 | 捕获第三方应用 Toast 通知，以 Alert 形式展示 |
| 网络状态 | 监控 WiFi/蓝牙连接状态变化，自动弹出提醒 |
| 文件拖放 | 支持将文件拖放到悬浮球上暂存 |
| 天气插件 | 和风天气 API + 几何矢量图标 + 日夜自动切换 |
| 系统监控 | 电量、充电状态监控 |
| 弹簧物理 | 所有尺寸/透明度/滚动过渡均使用质量-弹簧-阻尼模型 |
| 优先级告警 | P0-P3 四级告警队列，支持高优先级中断 |
| 全屏防打扰 | 检测全屏游戏/应用时自动隐藏 |
| 多Island模式 | IslandManager 支持 Main/Mini/Pomodoro/Stats 四种 Island 类型 |

## 4. 核心架构解析

### 4.1 入口点与初始化流程

**文件**: `cpp/main.cpp`

```cpp
int WINAPI wWinMain(HINSTANCE hInstance, HINSTANCE hPrevInstance,
                    PWSTR pCmdLine, int nCmdShow)
{
    // 1. 初始化 OLE（拖放功能需要）
    OleInitialize(nullptr);

    // 2. 设置 DPI 感知（Per-Monitor V2）
    SetProcessDpiAwarenessContext(DPI_AWARENESS_CONTEXT_PER_MONITOR_AWARE_V2);

    // 3. 创建并运行 DynamicIsland 实例
    DynamicIsland app;
    app.Initialize();
    return app.Run();
}
```

**初始化序列**:
1. `DynamicIsland::Initialize()` — 创建窗口、初始化渲染引擎、加载 config.ini、启动各监控线程、注册 EventBus 订阅、创建托盘图标、启动 ~16ms 动画定时器
2. `DynamicIsland::Run()` — 进入 Win32 消息循环

> **注意**：WinRT 的初始化（`winrt::init_apartment`）在 MediaMonitor / NotificationMonitor 内部各自管理，采用 `single_threaded` 公寓类型。

### 4.2 窗口管理与消息循环

**文件**: `cpp/DynamicIsland.cpp` / `h/DynamicIsland.h`

**关键消息处理**：

| 消息 | 处理逻辑 |
|------|---------|
| `WM_CREATE` | 初始化渲染引擎、启动监控线程 |
| `WM_DESTROY` | 清理资源、退出消息循环 |
| `WM_TIMER` | 驱动弹簧物理更新（~16ms），全屏检测（OPT-01） |
| `WM_MOUSEMOVE` | 悬停检测、天气图标动画触发 |
| `WM_LBUTTONDOWN` | 命中检测（按钮/进度条/展开区域） |

| `WM_MOUSELEAVE` | 鼠标离开，收起 Expanded 模式 |
| `WM_HOTKEY` | 全局热键处理（F10 / Ctrl+Alt+Shift+S） |
| `WM_POWERBROADCAST` | 电源事件（睡眠/唤醒/电量变化） |
| `WM_DPICHANGED` | DPI 变化，重新创建渲染资源 |
| `WM_UPDATE_ALBUM_ART` | 从文件加载专辑封面（跨线程） |
| `WM_UPDATE_ALBUM_ART_MEMORY` | 从内存加载专辑封面（WinRT 缩略图） |
| `WM_SHOW_ALERT` | 触发告警显示，入队优先级调度 |
| `WM_UPDATE_LYRICS` | 更新当前歌词行 |
| `WM_DRAG_ENTER/LEAVE/DROP_FILE` | 文件拖放处理 |

### 4.3 渲染引擎

**文件**: `cpp/RenderEngine.cpp` / `h/RenderEngine.h`

渲染引擎基于 **Direct2D 1.1** 和 **DirectComposition**，使用分层渲染表面实现透明窗口。

#### 初始化流程

```cpp
bool RenderEngine::Initialize(HWND hwnd, int canvasWidth, int canvasHeight)
{
    // 1. D3D11CreateDevice → ID3D11Device
    // 2. D2D1CreateFactory → ID2D1Factory1
    // 3. IDXGIDevice → ID2D1Device → ID2D1DeviceContext
    // 4. DCompositionCreateDevice → IDCompositionDevice
    // 5. IDCompositionTarget (绑定到 HWND)
    // 6. IDCompositionSurface → 渲染表面
    // 7. 预创建所有画笔（WiFi绿、BT蓝、电量红等）
    // 8. IDWriteFactory → 字体资源
}
```

#### 渲染入口

```cpp
void RenderEngine::DrawCapsule(const RenderContext& ctx);
```

根据 `ctx.displayMode` 分发到具体组件：

| 模式 | 渲染路径 |
|------|---------|
| `Idle` | 渲染背景胶囊 + 时间/天气 |
| `MusicCompact` | `MusicPlayerComponent::Draw(Compact)` |
| `MusicExpanded` | `MusicPlayerComponent::Draw(Expanded)` |
| `Alert` | `AlertComponent::Draw()` |
| `Volume` | `VolumeComponent::Draw()` |

#### TextLayout 缓存

```cpp
// 避免每帧重复创建 IDWriteTextLayout
std::unordered_map<std::wstring, ComPtr<IDWriteTextLayout>> m_textLayoutCache;
```

### 4.5 弹簧物理动画系统

**文件**: `h/Spring.h`

使用质量-弹簧-阻尼模型（Semi-implicit Euler 积分）：

```cpp
class Spring
{
public:
    // 工厂方法
    static Spring CreateBouncy();   // 高张力，弹性感强
    static Spring CreateSmooth();   // 低张力，平滑过渡
    static Spring CreateDefault();  // 默认参数

    void SetTarget(float target);
    bool Update(float dt);          // 返回 true 表示仍在运动
    float GetValue() const;

private:
    float m_value{0};
    float m_target{0};
    float m_velocity{0};
    float m_stiffness{100.f};  // 弹簧刚度
    float m_damping{10.f};     // 阻尼系数
    float m_mass{1.f};         // 质量
};
```

**物理积分**:
```
F = -k * (x - target) - c * v
a = F / m
v += a * dt        // Semi-implicit Euler
x += v * dt
```

**应用场景**：

| Spring 实例 | 控制对象 |
|------------|---------|
| `m_widthSpring` | 岛屿宽度过渡 |
| `m_heightSpring` | 岛屿高度过渡 |
| `m_alphaSpring` | 透明度淡入/淡出 |
| `m_scrollOffsetX` | 标题/艺术家水平滚动 |
| `m_lyricsScrollY` | 歌词垂直滚动（OPT-05） |

#### 架构

class MediaMonitor

#### 智能轮询（OPT-02）

```
播放中    → 轮询间隔 1000ms
暂停中    → 轮询间隔 5000ms
无会话    → 轮询间隔 10000ms
```

```

  → 读取字节数组

```

session.TryPlayAsync();

### 5.3 LyricsMonitor 歌词监控

**文件**: `cpp/LyricsMonitor.cpp` / `h/LyricsMonitor.h`（~600 行）

从**网易云音乐 API** 获取歌词，解析 LRC 格式并与播放进度同步。

#### LRC 格式

```
[00:12.34]这是一行歌词
[01:05.30]副歌开始
```

#### 数据结构

```cpp
struct LyricLine {
    int64_t timestampMs;   // 时间戳（毫秒）
    std::wstring text;     // 歌词文本
};

struct LyricData {
    std::wstring text;       // 当前行歌词
    int64_t currentMs;       // 当前行开始时间
    int64_t nextMs;          // 下一行开始时间
    int64_t positionMs;      // 当前播放位置
};
```

#### 获取流程（后台异步）

1. 以歌曲名 + 艺术家为 key 搜索网易云 API（URL 编码中文）
2. 获取歌曲 ID
3. 获取 LRC 歌词文本
4. 解析 `[mm:ss.xx]` 时间戳 → `std::vector<LyricLine>`
5. 通过 `WM_UPDATE_LYRICS` 传给主线程
6. `MusicPlayerComponent` 用弹簧物理滚动显示当前行（OPT-05）

#### 歌词滚动（OPT-05）

- 计算当前行在歌词列表中的索引
- 用 `Spring::SetTarget()` 设置目标滚动位置
- 末端减速：接近目标时降低弹簧张力

## 7. 事件通信机制

### 7.1 EventBus 事件总线

**文件**: `h/EventBus.h`

线程安全的单例发布/订阅系统：

```cpp
class EventBus
{
public:
    static EventBus& GetInstance();

    int Subscribe(EventType type, std::function<void(const Event&)> callback);
    bool Unsubscribe(EventType type, int id);
    void Publish(const Event& event);

private:
    std::mutex m_mutex;
    std::map<EventType, std::vector<std::pair<int, Callback>>> m_subscribers;
};
```

#### 支持的事件类型

```cpp
enum class EventType {
    PowerChange,
    MediaSessionChanged,
    MediaPlaybackStateChanged,
    MediaMetadataChanged,
    MediaProgressChanged,
    SystemTimeChanged,
    VolumeChanged,
    WindowStateChanged,
    NotificationArrived,
    NetworkStatusChanged,
    BluetoothStatusChanged,
    Custom
};
```

#### 使用模式

```cpp
// 订阅
int id = EventBus::GetInstance().Subscribe(
    EventType::MediaPlaybackStateChanged,
    [this](const Event& e) { /* 处理 */ }
);

// 发布
EventBus::GetInstance().Publish(Event{
    EventType::MediaPlaybackStateChanged,
    /* payload */
});

// 取消订阅（在析构时调用）
EventBus::GetInstance().Unsubscribe(EventType::MediaPlaybackStateChanged, id);
```

> **线程安全**：`Publish()` 可从任意后台线程调用，内部加 `std::mutex` 保护订阅者列表；回调执行在发布方线程，需注意 UI 操作应通过 `PostMessage` 转回主线程。

## 9. 通知优先级系统（OPT-03）

**文件**: `h/Messages.h`, `cpp/DynamicIsland.cpp`

告警按优先级入队，支持 P0 级中断当前显示：

```cpp
enum AlertPriority {
    PRIORITY_P0_CRITICAL = 0,  // 可中断当前任何告警
    PRIORITY_P1_IMMEDIATE = 1, // 立即插队（聊天类应用）
    PRIORITY_P2_MEDIA = 2,     // 媒体控制事件
    PRIORITY_P3_BACKGROUND = 3 // 静默后台状态
};
```

```cpp
// 告警入队
void DynamicIsland::EnqueueAlert(const AlertInfo& alert)
{
    // P0 直接中断当前告警
    if (alert.priority == PRIORITY_P0_CRITICAL) {
        ProcessAlertWithPriority(alert);
        return;
    }
    m_alertQueue.push(alert);
}

// 当前告警结束后处理下一个
void DynamicIsland::ProcessNextAlert()
{
    if (!m_alertQueue.empty()) {
        auto next = m_alertQueue.front();
        m_alertQueue.pop();
        ShowAlert(next);
    }
}
```
