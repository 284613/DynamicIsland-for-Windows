# 多岛轮播系统 - 设计文档

## 概述

将 DynamicIsland 改造为任务轮播式岛屿系统，通过点击切换显示不同任务内容，无任务时显示用户自定义功能菜单。

## 交互模型

### 状态定义

| 状态 | 显示 |
|------|------|
| 0 任务 | Mini 主岛 + 时间 |
| 1 任务 | 主岛直接显示任务内容 |
| 2+ 任务 | 轮播界面：左圆圈 ← 主岛(当前任务) → 右圆圈 |

### 轮播交互

```
[○ 任务A]  [主岛: 任务B内容]  [○ 任务C]
     ↑点击切换到任务A
            ↑点击切换到任务C
                         ↑点击切换到任务A
```

- **点击左圆圈**：主岛显示上一任务
- **点击主岛**：主岛显示下一任务
- **点击右圆圈**：主岛显示下一任务
- **左右滑动**：支持鼠标滑过切换

## 任务检测（自动）

| 任务 | 检测条件 | 主岛显示内容 |
|------|----------|--------------|
| 音乐 | Spotify/QQ音乐等播放中 | 专辑封面+播放控制 |
| 番茄钟 | 计时器运行中 | 倒计时+进度条 |
| 通知 | 有新通知待处理 | 通知图标+应用名 |
| 文件传输 | LAN传输进行中 | 传输进度+速度 |
| 歌词 | 音乐播放且有歌词 | 歌词文字 |

## 无任务功能菜单

当检测到 0 个任务时，主岛下方展开功能菜单（可自定义）：

```
     [Mini 主岛]
        时间
    ──────────────
    [时钟] [天气] [日历]
    [截图] [计算器] [系统监控]
```

### 可选功能（用户配置）

- 时钟（数字/模拟）
- 天气（当前温度+状态）
- 日历（当前日期+周几）
- 快捷操作：截图、录屏、计算器
- 系统监控：CPU、内存、网络速度

## 架构设计

### 核心组件

**IslandCarousel - 轮播管理器**
- 管理任务列表和当前索引
- 处理点击切换逻辑
- 协调主岛和圆圈的渲染

**TaskDetector - 任务检测器**
- 监听各模块状态变化
- 维护活动任务列表
- 通知 carousel 更新

**FunctionMenu - 功能菜单**
- 用户自定义功能配置
- 渲染功能网格
- 处理功能点击事件

### 数据流

```
TaskDetector 检测到任务变化
    ↓
IslandCarousel 更新任务列表
    ↓
渲染层：
  - 0任务 → Mini主岛 + FunctionMenu
  - 1任务 → 主岛显示任务内容
  - 2+任务 → 圆圈 + 主岛(当前任务) + 圆圈
```

## 配置存储

`settings.json` 结构：

```json
{
  "island": {
    "autoDetect": true,
    "functions": ["clock", "weather", "screenshot", "monitor"]
  }
}
```
