# 下一批功能路线图

## Summary

本路线图记录优化收口后的下一批功能，采用工程排期风格，目标是让后续开发可以直接按优先级拆分任务并进入实现。排序原则：

- 先补“历史中心”和“上下文联动”，提升长期可用性
- 再增强文件副岛与 TODO 的日常效率能力
- 最后补番茄钟统计类功能

## P0

### 1. 任务历史中心
- 目标：为 Todo / Pomodoro / Agent 提供统一的历史入口，能回看最近任务、完成记录和中断点
- 实现入口：`DynamicIsland.cpp` 状态入口、`src/components/AgentSessionsComponent.cpp`、新增本地 history store
- 验收条件：
  - 至少展示最近 20 条历史记录
  - 区分来源：Todo / Pomodoro / Agent
  - 支持按时间倒序查看
  - 应用重启后历史仍可读取

### 2. 通知历史中心
- 目标：把当前“只弹不留”的通知流补成可回看的通知历史面板
- 实现入口：`NotificationMonitor.cpp`、告警/通知组件、设置页入口或 Agent/通知扩展面板
- 验收条件：
  - 保留最近 50 条通知
  - 每条通知包含应用名、标题摘要、到达时间
  - 支持清空历史
  - 不影响现有实时弹窗逻辑

### 3. Agent 面板工作区 / 分支 / 构建状态联动
- 目标：让 Agent 面板不只看会话，还能看到当前工作区上下文
- 实现入口：`AgentSessionMonitor.cpp`、`AgentSessionsComponent.cpp`、本地 git/build 状态采集
- 验收条件：
  - 每个 Agent 会话可显示工作目录标签
  - 当前选中会话可显示分支名
  - 当前选中会话可显示最近一次构建结果或“未构建”
  - 数据刷新不会明显增加 idle 态 CPU 占用

## P1

### 4. 文件副岛增强：多选 / 置顶 / 自动清理
- 目标：把文件副岛从临时暂存升级成轻量工作台
- 实现入口：`FileStashStore.h`、`FilePanelComponent.cpp`、`FileSecondaryInput.cpp`
- 验收条件：
  - 支持多选操作
  - 支持置顶文件
  - 支持基于数量或时间的自动清理
  - 启动恢复逻辑与新字段兼容

### 5. TODO 截止时间与提醒
- 目标：让 TODO 从纯列表升级为带时效性的任务入口
- 实现入口：`TodoStore.cpp`、`TodoComponent.cpp`、告警/通知调度
- 验收条件：
  - Todo 可设置截止时间
  - 到期前可触发提醒
  - 紧凑态可识别最近到期任务
  - 老的 `todos.json` 数据能兼容升级

## P2

### 6. 番茄钟统计
- 目标：补齐番茄钟的长期反馈能力，形成可回顾的数据面板
- 实现入口：`PomodoroComponent.cpp`、新增统计 store、设置或独立统计面板
- 验收条件：
  - 统计每日完成次数与总专注时长
  - 至少支持最近 7 天视图
  - 重启后统计数据可保留
  - 完成一次番茄后统计会实时更新

## Acceptance

- 路线图中的 6 项功能都有明确目标、实现入口、验收条件
- 实现顺序固定为：P0 → P1 → P2
- 后续新增功能若不属于上述 6 项，应另开新路线图，不混写到本文件
